Processing Methodology

This document explains how the Voynich manuscript transcription is processed to generate character transition matrices. The processing pipeline transforms raw transcription data into normalized character sequences, then counts transitions between characters to reveal statistical patterns.

Pipeline Overview

Step 1: Parse Transcription

Load the IVTFF (Intermediate Voynich Transcription File Format) transcription and extract page metadata including folio ID, Currier language (A/B), scribal hand (1-5), illustration type, and quire number.

Step 2: Character Normalization

Transform raw transcription characters into a normalized form by collapsing multi-character sequences (bigrams) into single tokens and handling repeated i/e characters according to the configured mode.

Step 3: Boundary Token Insertion

Insert special boundary tokens to mark word, line, paragraph, and page boundaries. This allows analysis of character positions within structural units.

Step 4: Transition Counting

For each page, count every character pair (bigram) that appears in sequence. This produces a transition matrix showing how often each character follows each other character.

Step 5: Aggregation

Combine page-level transition counts into aggregated datasets by filtering on metadata (language, hand, section type). Compute both raw counts and row-normalized probabilities.

Character Substitutions

Bigram Collapsing

Multi-character sequences in the EVA transcription alphabet are collapsed into single tokens to simplify analysis. This is applied in order from longest to shortest to handle overlapping patterns correctly.

i/e Sequence Handling

The manuscript contains many repeated 'i' and 'e' characters. These are mapped to distinct tokens based on their count. In "distinct" mode, each length maps to a unique token. In "collapsed" mode, all lengths map to a single token (I or E).

Reading the Transition Matrix

In the visualization, rows represent the source character and columns represent the target character. Each cell shows how likely (or how often) the target character follows the source character.

For example, a bright cell at row "k" and column "o" indicates that "o" frequently follows "k" in the text. Row-normalized probabilities sum to 1.0 across each row, showing the conditional probability P(target | source).
