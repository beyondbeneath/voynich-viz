# Transition Matrix

This codebase computes per-glyph transition probabilities (Markov chain style) from the Voynich manuscript transcription. The Python pipeline parses the IVTFF transcription format, normalizes characters by collapsing common bigrams into single tokens (ch→C, sh→S, cth→T, etc.) and handling i/e sequences (either distinctly as ii→2, iii→3 or collapsed as ii→I), then counts character-pair transitions on each page. Crucially, it tracks transitions at structural boundaries—word, line, paragraph, and page start/end markers—allowing analysis of positional patterns (e.g., which characters typically begin lines or end words). Per-page counts are then aggregated by various filters: Currier language (A/B), scribal hand (1/2), illustration type (herbal, zodiac, biological, pharmaceutical, astronomical, cosmological, text-only), and cross-filtered combinations (e.g., herbal + Language A). All results are output as JSON.

The accompanying browser-based visualizer displays these transition probabilities as interactive heatmap matrices. Users can switch between aggregations, toggle between raw counts and probabilities, apply log scaling for rare transitions, choose different color scales, and optionally render characters in the Voynich EVA font. The matrix rows represent "from" characters and columns represent "to" characters, with boundary markers shown as separate categories, enabling visual inspection of how character transition patterns vary across manuscript sections, languages, and scribal hands.
